## Введение

Данная практическая работа направлена на получение практических навыков по написанию ROS2 пакетов с целью дальнейшего создания сложных автоматических алгоритмов управления.

В качестве начальной задачи будет написание ROS2 пакета для реализации алгоритма возврата домой по информации о расстоянии до дома.

## Выполнение практической работы

### Этап 1 - Подготовка пакетов

В качестве примером будем рассматривать пакеты ros2_mavros_takeoff и ros2_mavros_waypoint_mission.

Для написания пакетов используется официальное [руководство](https://docs.ros.org/en/jazzy/Tutorials/Beginner-Client-Libraries/Creating-Your-First-ROS2-Package.html).

После написания пакетов необходимо осуществить их сборку. Выполнять команды необходимо из директрии, в которой расположены пакеты.

```bash
 colcon build --packages-select ros2_mavros_waypoint_mission
```

```bash
colcon build --packages-select ros2_mavros_takeoff
```

После компиляции необходимо в каждом терминале выполнить скрипт для подключения собранных пакетов к ROS2. При включении новых терминалов необходимо также выполнять данную команду из директории с собранными пакетами.

```bash
source install/setup.bash
```

### Этап 2 - Запуск пакетов

После сборки пакетов и подключения данных пакетов, можно запускать симулятор. Производим запуск симулятора Gazebo и Ardupilot аналогично Практической работе №2.

#### В первом терминале введите команду запуска симулятора Gazebo:

```bash
gz sim -v4 -r iris_runway.sdf
```

#### Во втором терминале введите последовательность команд:

1) Переход в директорию со скриптом запуска SITL:

```bash
cd ardupilot/Tools/autotest/
```

2. Запуск скрипта SITL

```bash
python3 sim_vehicle.py -D -v ArduCopter -f JSON --add-param-file=$HOME/ardupilot_gazebo/config/gazebo-iris-gimbal.parm --console --map
```

Дождитесь окончания компиляции и открытия всех окон SITL Ardupilot.

#### В третьем терминале запустите MAVROS на порту 5762:

```bash
ros2 launch mavros apm.launch fcu_url:=tcp://127.0.0.1:5762
```

#### В четвертом терминале вызовите сервис для принудетельной установки частоты публикации топиков:

```bash
ros2 service call /mavros/set_stream_rate mavros_msgs/srv/StreamRate "{stream_id: 0, message_rate: 10, on_off: true}"
```

#### В пятом терминале проверьте список публикуемых топиков:

```bash
ros2 topic list
```

и проверьте поток данных с топика `/mavros/local_position/pose`

```bash
ros2 topic echo /mavros/local_position/pose
```

#### Шестой терминал используете для запуска написанных нод.

Нода для арма и взлета:

```bash
ros2 run ros2_mavros_takeoff arm_takeoff
```

Нода для полета по точкам:

```bash
ros2 run ros2_mavros_waypoint_mission waypoint_mission --ros-args --params-file ros2_mavros_waypoint_mission/config/waypoints.yaml
```

Обратите внимание на запуск второй ноды. При запуске мы передаем файл с параметрами, в нашем случае с координатам точек полета. Это необходимо для того, чтобы при частом изменении параметров не пересобирать пакет.

### Этап 3 - Практическое задание

Во первых необходимо дописать пакет ros2_mavros_distance_home. Данный пакет создает топик и публикует в него расстояние от дрона до  точки "home".

Далее вам необходимо написать ROS2 пакет для реализации алгоритма возврата домой по информации о расстоянии до дома.

При создании пакета, ориентируйтесь на уже готовые предоставленные пакеты.

В данном пакете нода должны быть подписана на топик, в который публикуется информация о расстоянии от дрона до точки "home".

Исходя из получаемного расстояния, следуя уже известной нам логике работы алгоритма (см. Практическую работу №3), необходимо задавать положение дрона через топик `/mavros/setpoint_attitude/attitude`(обратите внимание, что управление через данный топик осуществляется с помощью кватернионов). Вы также можете использовать и другие топики/сервисы для управления дроном.
