## Введение

Данная практическая работа направлена на получение практических навыков модификации исходного кода открытого программного обеспечения на примере автопилота Ardupilot.

В качестве модификации будет внесение дополнительного функционала, а именно - написание функции для нахождения Евклидова расстояния от дрона до точки "home".

## Выполнение практической работы

### Этап 1 - Подготовка окружения

В первую очередь необходимо выяснить, присутствует ли в уже имеющимся коде необходимый нам функционал. С одной стороны, данное действие необходимо, чтобы не "изобретать велосипед". С другой стороны, исследуя существующий код, можно во-первых найти функционал, который можно использовать для написания собственного кода, во-вторых, найти функционал, выполняющий задачу, приближенную к нашей и модифицировать его под наши требования. Всё это в конечном счёте упрощает ведение разработки.

После анализа архитектуры программы и поиска функционала, приближенного к нашей задаче, необходимо определить, в какое место архитектуры будет интегрироваться наш модифицированный код. В первую очередь будем ориентироваться на правило "подобный функционал должен находиться среди подобного". Главная причина использования даннного правила заключается в необходимости понимания кода другими людьми (или вами же через месяц).

Исходя из вышесказанного, определим в какое место будет интегрироваться наш код для выполнения поставленной задачи написания функции для нахождения Евклидоварасстояния от дрона до точки "home".

Проведя анализ существующего кода, можно заметить, что в библиотеке "AP_Common" файл "Location.cpp" содержит функцию "get_distance", выполняющую задачу, похожую на поставленную нами.  Во-перых, мы можем использовать данную функцию для написания нашей функции. Во-вторых, код нашей функции необходимо расположить рядом с кодом уже имеющейся функции.

Таким образом, мы будем модифицировать следующие файлы:

`/ardupilot/libraries/AP_Common/Location.cpp`

`/ardupilot/libraries/AP_Common/Location.h`

Кроме того, поскольку предполагается использование нового функционала в lua-скриптах, необходимо внести изменения и в следующие файлы, определяющие функции, используемые в lua-скриптах:

`/ardupilot/libraries/AP_Scripting/generator/description/bindings.desc`

`/ardupilot/libraries/AP_Scripting/docs/docs.lua`

В заголовочном файле `/ardupilot/libraries/AP_Common/Location.h` на строке 106 определим новую функцию:

```cpp
ftype get_distance_3d(const Location &loc2) const WARN_IF_UNUSED;
```

Включаем данную функцию в набор скриптов lua через добавление на 14 строку в файле `/ardupilot/libraries/AP_Scripting/generator/description/bindings.desc` кода:

```cpp
userdata Location method get_distance_3d float Location
```

А так же на 1158 строку в файле `/ardupilot/libraries/AP_Scripting/docs/docs.lua`:

```cpp
function Location_ud:get_distance_3d(loc) end
```
