## Введение

Данная практическая работа направлена на получение практических навыков визуализации телеметрических данных летательного аппарата в среде RViz2.

Цель практической работы — запустить RViz2 и настроить отображение следующих данных, получаемых через MAVROS:

* текущее положение и ориентация дрона (поза);
* траектория полёта (путь);

Задачи:

1. Запустить симулируемую среду (Gazebo + Ardupilot SITL);
2. Запустить MAVROS и убедиться в корректной публикации топиков;
4. Запустить RViz2 и настроить необходимые плагины отображения;
5. Написать ROS2 пакет, публикующий дополнительные визуализационные маркеры.

> **Важно:** Связка ROS2 + Ardupilot + MAVROS имеет ряд известных особенностей. Каждая из них описана в соответствующих этапах данной работы. Внимательно читайте предупреждения.

## Выполнение практической работы

### Этап 1 - Запуск симуляционного окружения

Запуск производится аналогично Практической работе №4. Откройте шесть терминалов (используйте terminator).

#### Терминал 1 — Запуск симулятора Gazebo:

```bash
gz sim -v4 -r iris_runway.sdf
```

#### Терминал 2 — Запуск Ardupilot SITL:

```bash
cd ardupilot/Tools/autotest/
python3 sim_vehicle.py -D -v ArduCopter -f JSON --add-param-file=$HOME/ardupilot_gazebo/config/gazebo-iris-gimbal.parm --console --map
```

Дождитесь окончания компиляции и открытия всех окон SITL Ardupilot.


#### Терминал 3 — Запуск MAVROS:

```bash
ros2 launch mavros apm.launch fcu_url:=tcp://127.0.0.1:5762
```



#### Терминал 4 — Установка частоты публикации топиков:

> **Проблема — частота публикации топиков.**
> После запуска MAVROS многие топики (в том числе `/mavros/local_position/pose`) могут не публиковаться или публиковаться с частотой 0 Гц. Это штатное поведение: Ardupilot по умолчанию не стримит все MAVLink-сообщения. Для принудительной установки частоты публикации необходимо вызвать сервис (см. Терминал 4).

```bash
ros2 service call /mavros/set_stream_rate mavros_msgs/srv/StreamRate "{stream_id: 0, message_rate: 10, on_off: true}"
```

Значение `stream_id: 0` означает включение всех потоков данных. `message_rate: 10` — 10 Гц.


#### Терминал 5 — Проверка публикуемых топиков:

```bash
ros2 topic list
```

Убедитесь, что в списке присутствуют следующие топики:

```
/mavros/local_position/pose
/mavros/local_position/odom
/mavros/global_position/global
/mavros/imu/data
/mavros/state
```

Проверьте поток данных с топика позиции:

```bash
ros2 topic echo /mavros/local_position/pose
```

Проверьте топик состояния:

```bash
ros2 topic echo /mavros/state
```

Поле `connected` должно иметь значение `true`.


### Этап 2 - Запуск и настройка RViz2

#### Терминал 6 — Запуск RViz2:

```bash
ros2 run rviz2 rviz2
```

#### Настройка фиксированного фрейма (Fixed Frame):

В панели `Global Options` в левой части RViz2 измените значение поля `Fixed Frame` на:

```
map
```


#### Добавление отображения положения дрона (Pose):

1. Нажмите кнопку `Add` в левой нижней части панели `Displays`.
2. Выберите тип `Pose` и нажмите `OK`.
3. В настройках добавленного плагина укажите топик: `/mavros/local_position/pose`.
4. Измените `Shape` на `Arrow` для отображения в виде стрелки.

> **Важно — Reliability Policy.**
> При добавлении любого плагина в RViz2 в настройках топика присутствует поле `Reliability Policy`. Его необходимо выставить в значение **`System Default`**. При любом другом значении (`Reliable` или `Best Effort`) RViz2 может не получать данные от MAVROS, поскольку политика QoS подписчика должна совпадать с политикой, используемой издателем.

#### Добавление отображения траектории (Path):

1. Нажмите `Add` → выберите тип `Path`.
2. Топик `Path` должен получать тип `nav_msgs/Path`.

> **Проблема — тип сообщения для Path.**
> MAVROS не публикует топик типа `nav_msgs/Path` напрямую. Для отображения траектории (пути) необходимо
> написать ROS2 ноду, подписывающуюся на `/mavros/local_position/pose` (тип `PoseStamped`) и накапливающую точки в массив, публикуя его в отдельный топик типа `nav_msgs/Path` (реализуется в Этапе 3 в качестве задания);


#### Добавление отображения одометрии (Odometry):

1. Нажмите `Add` → выберите тип `Odometry`.
2. Укажите топик: `/mavros/local_position/odom`.
3. В настройках можно отрегулировать `Keep` (количество отображаемых стрел) и масштаб.


#### Сохранение конфигурации RViz2:

После настройки всех плагинов сохраните конфигурацию через меню:

`File` → `Save Config As`

Сохраните файл с расширением `.rviz` в директорию вашего ROS2 пакета.

#### Задание 2

В рамках данного задания необходимо написать ROS2 пакет `ros2_rviz_drone_viz`, который будет публиковать дополнительные данные для отображения в RViz2.

Для написания пакета используйте официальное [руководство](https://docs.ros.org/en/jazzy/Tutorials/Beginner-Client-Libraries/Creating-Your-First-ROS2-Package.html) и уже имеющиеся пакеты из Практической работы №4 в качестве примера.

#### Нода `path_publisher`

Нода должна:

1. Подписаться на топик `/mavros/local_position/pose` (тип `geometry_msgs/PoseStamped`);
2. Накапливать принятые позиции в список (максимум 500 точек, при превышении — удалять старые);
3. Публиковать накопленный путь в топик `/drone/path` (тип `nav_msgs/Path`) с частотой 5 Гц.

Пример структуры ноды:

```python
import rclpy
from rclpy.node import Node
from geometry_msgs.msg import PoseStamped
from nav_msgs.msg import Path

class PathPublisher(Node):
    def __init__(self):
        super().__init__('path_publisher_node')
        # Подписка на позицию дрона
        self.pose_sub = self.create_subscription(
            PoseStamped,
            '/mavros/local_position/pose',
            self.pose_callback,
            10)
        # Публикация пути
        self.path_pub = self.create_publisher(Path, '/drone/path', 10)
        # Таймер для публикации
        self.timer = self.create_timer(0.2, self.publish_path)

        self.path_msg = Path()
        self.path_msg.header.frame_id = 'map'
        self.max_poses = 500

    def pose_callback(self, msg):
        self.path_msg.poses.append(msg)
        if len(self.path_msg.poses) > self.max_poses:
            self.path_msg.poses.pop(0)

    def publish_path(self):
        self.path_msg.header.stamp = self.get_clock().now().to_msg()
        self.path_pub.publish(self.path_msg)

def main(args=None):
    rclpy.init(args=args)
    node = PathPublisher()
    rclpy.spin(node)
    node.destroy_node()
    rclpy.shutdown()
```

> Фрейм `header.frame_id` в сообщении `Path` должен совпадать с `Fixed Frame`, выбранным в RViz2. Если MAVROS публикует данные в фрейме `map`, то и в сообщении `Path` должно быть указано `frame_id = 'map'`. Несовпадение фреймов приведёт к тому, что траектория не будет отображена в RViz2 (или будет отображена некорректно).


#### Сборка пакета:

#### Запуск ноды:

```bash

```

После запуска ноды добавьте в RViz2 плагин `Path` и укажите топик `/drone/path`.

Выполните полёт "Возврат домой" из Практической работы №4 и продемонстрируйте отображение траектории полёта в RViz2.

### Этап 4 - Справочник известных проблем

В данном разделе собраны типовые проблемы, возникающие при работе со связкой ROS2 + Ardupilot + MAVROS, и способы их диагностики.

#### Проблема: MAVROS не подключается к SITL

**Симптомы:** В терминале MAVROS нет сообщения `Got HEARTBEAT, connected`.

**Возможные причины и решения:**

1. SITL ещё не завершил компиляцию — дождитесь полного запуска.
2. Неверный порт — убедитесь, что используется `tcp://127.0.0.1:5762`, а не UDP.
3. MAVProxy уже занял порт — перезапустите SITL и MAVROS.

#### Проблема: Топики MAVROS есть, но данные не поступают (частота 0 Гц)

**Симптомы:** `ros2 topic hz /mavros/local_position/pose` показывает 0.

**Решение:** Вызвать сервис `set_stream_rate` (Терминал 4 данной практики).

#### Проблема: RViz2 не отображает данные, хотя топик публикуется

**Симптомы:** Плагин добавлен, топик выбран, `ros2 topic echo` показывает данные — но в RViz2 ничего не отображается.

**Причина:** Несовпадение политики QoS между издателем (MAVROS) и подписчиком (RViz2).

**Решение:** В настройках каждого плагина в RViz2 найдите поле `Reliability Policy` и выставьте его в значение **`System Default`**. Это позволяет RViz2 автоматически согласовать политику QoS с издателем.

#### Проблема: Высокая нагрузка на CPU при работе RViz2

**Симптомы:** Система тормозит, большое потребление CPU.

**Решение:**

1. Уменьшите частоту публикации данных (измените `message_rate` в `set_stream_rate` до 5 Гц).
2. Уменьшите `max_poses` в ноде `path_publisher`.
3. Уменьшите частоту обновления RViz2 в настройках `Global Options` → `Frame Rate`.
